# Roadmap

Implementation is committed directly to `main` in coherent conventional commits. Public brand submissions remain pull-request based.

## 1. Workspace

Set up strict TypeScript, linting, tests, package boundaries, and release tooling.

## 2. Schema and assets

Implement runtime schemas, cross-record validation, hardened SVG parsing, deterministic sanitation, previews, and visual diffs.

## 3. Canonical data

Add high-value brands with reviewed sources, beginning with developer platforms, cloud providers, social platforms, payments, and common integrations.

## 4. Core and package

Build shared search/resolution/selection logic and publish the new `open-brands` package.

## 5. CDN and API

Publish immutable R2 releases, implement public API routes, OpenAPI, transformations, caching, and quotas.

## 6. Website and packs

Build the explorer, brand pages, collections, local cart, and asynchronous pack generation.

## 7. Contributions

Add GitHub authentication, private uploads, validation jobs, bot-created pull requests, required CI, and moderation controls.

## 8. Production

Deploy Pages, Workers, R2, Queues, D1, Durable Objects, domains, observability, npm publishing, and rollback procedures.
